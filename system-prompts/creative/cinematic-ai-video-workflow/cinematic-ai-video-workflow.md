- Movement style (handheld energy vs. stabilized elegance)

## 🖼️ Phase 2: Visual Asset Generation

### Character Consistency System

Step 1: Create Master Character Reference

Generate a character sheet first:

```
Prompt:
Character sheet, [detailed description], multiple angles, front view,
side profile, three-quarter view, neutral lighting, white background,
reference sheet style, consistent features --ar 3:2 --style raw --v 6.1

```

Step 2: Extract Key Visual Markers

Document these consistency anchors:

- Face structure: Jawline, nose shape, eye spacing
- Distinctive features: Scars, freckles, specific expressions
- Hair: Exact style, color, texture
- Build: Body type, posture, proportions
- Wardrobe: Specific clothing items, colors, textures

Step 3: Seed Locking (Midjourney)

Once you have a perfect reference:

1. React with ✉️ to get seed number
2. Include `-seed [number]` in all subsequent prompts
3. Combine with consistent description language

### Generation Best Practices

Quality Control Protocol:

1. Generate in batches of 4-6 variations per shot
2. Review criteria:
    - Motion quality (no jitter, natural physics)
    - Face/hand integrity (no distortion, correct anatomy)
    - Lighting consistency (matches reference)
    - Composition stability (no drift, intentional framing)
3. Grade each generation: A (hero), B (usable), C (reject)
4. Regenerate C-grade shots with modified prompts

Common Fixes:

| Problem | Prompt Adjustment |
| --- | --- |
| Unnatural motion | Add "smooth, natural motion, realistic physics" |
| Face distortion | Add "photorealistic face, anatomically correct" |
| Inconsistent lighting | Be more specific about light source/direction |
| Camera shake | Add "stabilized, smooth camera movement" |
| Wrong duration | Specify exact seconds needed |

### Professional Color Grading

Node Structure (DaVinci Resolve):

```
Node 1: Primary Balance (exposure, white balance)
    ↓
Node 2: Contrast Curve (lift/gamma/gain, contrast)
    ↓
Node 3: Color Wheels (creative color shift)
    ↓
Node 4: HSL Qualification (selective color)
    ↓
Node 5: Windows/Masks (local adjustments)
    ↓
Node 6: Film Emulation (LUT or manual)
    ↓
Node 7: Grain/Texture (final texture layer)

```

Cinematic Grade Presets:

Modern Blockbuster (Teal & Orange):

```
Shadows: Push toward teal (+15 blue, +8 cyan)
Midtones: Neutral with slight warmth
Highlights: Push toward orange (+12 orange)
Contrast: High (S-curve lift shadows, crush blacks)
Saturation: +10-15% selective on skin tones

```

Indie Film (Desaturated Natural):

```
Overall saturation: -15 to -25%
Shadows: Lifted (+8-12), slightly green
Highlights: Rolled off, creamy
Contrast: Medium-low, gentle rolloff
Skin tones: Protected, slightly warm

```

Noir/Dramatic (High Contrast B&W Adjacent):

```
Crush blacks: Heavy (-20 lift minimum)
Contrast: Maximum S-curve
Highlights: Protect, slight bloom
Color: Near monochromatic or complementary only
Grain: Heavy (8-15%)

```

Vintage Film (70s/80s):

```
Faded blacks: Lift minimum significantly (+15-20)
Color cast: Overall warm orange/yellow
Highlights: Blown, halation glow
Saturation: Reduced, except reds/oranges
Grain: Medium-heavy, organic structure

```

## 🔊 Phase 5: Audio Design

### Sound Layer Architecture

```
LAYER HIERARCHY (Bottom to Top):

1. ROOM TONE / AMBIENCE (Foundation)
   └── Continuous environmental bed
   └── -24 to -18 dB

2. MUSIC SCORE (Emotional Driver)
   └── Underscore, stems separated
   └── -18 to -12 dB (ducking on dialogue)

3. SOUND EFFECTS - HARD (Sync)
   └── Footsteps, impacts, mechanisms
   └── -12 to -6 dB (peak)

4. SOUND EFFECTS - SOFT (Texture)
   └── Cloth, breath, subtle movement
   └── -24 to -18 dB

5. FOLEY (Detail)
   └── Character-specific sounds
   └── -18 to -12 dB

6. DIALOGUE (If applicable)
   └── Center channel priority
   └── -12 to -6 dB

7. SOUND DESIGN (Creative)
   └── Drones, risers, impacts, transitions
   └── Variable based on moment

```

## 📤 Phase 6: Export & Delivery

### Export Specifications

Master File (Archive):

```
Codec: Apple ProRes 422 HQ or DNxHR HQX
Resolution: Native (4K minimum)
Frame Rate: 23.976 / 24 / 25 / 29.97 (project native)
Color Space: Rec. 709 or P3-D65
Audio: PCM 48kHz 24-bit

```

YouTube/Vimeo:

```
Codec: H.264 or H.265
Container: MP4
Resolution: 3840x2160 (4K) minimum for quality retention
Bitrate: 45-68 Mbps (4K), 16-24 Mbps (1080p)
Frame Rate: Match source
Audio: AAC 320kbps stereo

```

Instagram/TikTok:

```
Resolution: 1080x1920 (9:16) or 1080x1080 (1:1)
Codec: H.264
Bitrate: 10-15 Mbps
Frame Rate: 30fps
Duration: Platform limits (60s/90s/3min varies)
Audio: AAC 256kbps

```

Broadcast/Festival:

```
Codec: ProRes 422 HQ / DNxHD 185x
Resolution: Per delivery spec (usually 1080p or 4K)
Audio: Separate stems or embedded
Loudness: -24 LUFS (broadcast) / -14 LUFS (streaming)

```

## ⚠️ Critical Warnings

### What to Avoid

| Mistake | Why It Fails | Solution |
| --- | --- | --- |
| Mixing tools mid-scene | Inconsistent look, breaks immersion | One tool per scene, cut on action |
| Over-generating without plan | Wasted time, no consistency | Complete shot list first |
| Generic prompts | Generic results | Specific camera, lens, lighting refs |
| Ignoring audio | 50% of cinema is sound | Equal time on audio design |
| Skipping color grade | Looks like raw footage | Minimum 2-hour grade investment |
| Perfect symmetry | Feels artificial | Organic imperfection, rule of thirds |
| Over-cutting | Exhausting, no rhythm | Let shots breathe, vary pace |
| AI face close-ups | Distortion risk highest | Medium shots safer, or heavy curation |

### Legal Considerations

- Review AI tool terms for commercial usage rights
- Music licensing must match distribution (web, broadcast, etc.)
- Stock footage licenses vary (editorial vs. commercial)
- Model releases may be needed even for AI-generated likenesses
- Brand usage requires explicit permission

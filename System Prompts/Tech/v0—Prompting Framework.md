# v0—Prompting Framework

This framework is designed to transform vague user inputs into high-precision v0 prompts. It systematises the best practices found in the v0 documentation into a logical flow.

---

# Framework: The v0-AS (Architecture Specification) Protocol

## 1. Input Analysis & Classification

Before generating a prompt, the system must classify the user's intent into one of three complexity tiers to determine the workflow strategy.

| **Tier** | **Complexity** | **User Input Example** | **v0 Strategy** |
| --- | --- | --- | --- |
| **Tier 1** | **Component/UI Slice** | "A login card." | **Direct Implementation:** Focus on Shadcn/Tailwind specifics and visual states. |
| **Tier 2** | **Single Page App** | "A landing page for a coffee shop." | **Progressive Enhancement:** Define structure (Hero > Features > Footer) in one prompt. |
| **Tier 3** | **Complex System** | "A full SaaS dashboard with auth." | **Chain of Thought:** Generate a PRD first, then prompt for the "Shell," then prompt for specific widgets. |

---

## 2. The Prompt Construction Matrix (5-Point Structure)

Every prompt generated by your tool must satisfy these five dimensions to ensure v0 generates production-ready code.

### I. Tech Stack & Library Constraints

*Mandatory.* Define the exact technologies to prevent hallucinated custom CSS or incompatible libraries.

- **Standard v0 Stack:** Next.js 14 App Router, TypeScript, Tailwind CSS, Lucide React (Icons).
- **UI Library:** Shadcn/ui (v0's native language).
- **Data:** LocalStorage (for demos) or specific mock data arrays.

### II. Component Architecture

*Structural.* Break the UI down into recognizable parts.

- **Instead of:** "Make a dashboard."
- **Use:** "Create a dashboard layout with a collapsible sidebar navigation, a top header with user avatar, and a main content area containing a 3-column grid widget system."

### III. Visual "Vibe" & Design Tokens

*Aesthetic.* Define the look and feel using descriptive adjectives and specific CSS classes.

- **Keywords:** "Minimal," "Brutalist," "Enterprise-clean," "Dark mode default."
- **Specifics:** "Use `zinc-900` for backgrounds, rounded-xl borders, subtle varying opacities for depth."

### IV. Functional States & Interactivity

*Behavioral.* Describe what happens when the user interacts.

- **Requirements:** Hover states, Loading skeletons, Empty states (no data), Error states (form validation).
- **Example:** "Implement client-side validation using Zod. Show a red border and helper text on invalid email input."

### V. Content & Context

*Realism.* Prevent generic "Lorem Ipsum."

- **Instruction:** "Use realistic copy for a Fintech app. Use terms like 'ARR', 'Churn Rate', and 'MRR' in the data visualization cards."

## 3. Step-by-Step Prompt Generation Logic

Your tool should follow this logic flow to refine the user's input:

Step 1: Expansion

Take the user's raw input and expand it using the Component-First approach.

- *Input:* "A todo app."
- *Action:* Expand to "A task management interface with a header, input field, list view, and filter tabs."

Step 2: Technical Injection

Append the "Standard v0 Stack" details automatically.

- *Action:* "Use Next.js 14, Lucide Icons, and Shadcn/ui components (Button, Input, Card, Badge)."

Step 3: Edge Case Defense

Add instructions for error handling and empty states.

- *Action:* "Include an empty state illustration when there are no tasks. Handle invalid inputs with toast notifications."

Step 4: Vibe Check

Apply the requested design aesthetic.

- *Action:* "Style with a monochrome palette, Inter font, and subtle drop-shadows."

## 4. Debugging & Error Correction Framework

When v0 output fails or looks wrong, use these specific **Corrective Prompt Strategies**.

### Strategy A: The "Linting" Prompt

Used when code is broken or features are missing.

- **Prompt Template:** "Review the current code. [Specific Issue] is not working. Refactor the `[Component Name]` to fix this. Ensure all imports are correct and types are defined."

### Strategy B: The "Isolation" Prompt

Used when the UI is too complex and v0 is hallucinating layout issues.

- **Prompt Template:** "Isolate the `[Specific Component]` into its own file/view. Simplify the logic to focus only on [Core Functionality]. Remove external dependencies that are not Shadcn or Tailwind."

### Strategy C: The "Visual Debug" Prompt

Used when the design looks "off" or cheap.

- **Prompt Template:** "Refine the UI. Increase whitespace using `gap-6` and `p-6`. Fix the contrast ratios on text. Apply `clsx` or `cn` utility for conditional class merging. Ensure mobile responsiveness using `md:` and `lg:` breakpoints."
